# Glossary

> Definitions of terms used throughout the Shell documentation.

| Term | Definition |
| --- | --- |
| **5-gate auth chain** | The sequence of five authentication checks that must all pass before a terminal session starts: admin role, global toggle, agent cert valid, time window active, IP ACL. |
| **Admin** | The user who manages shell access — enables/disables agents, creates policies, connects to sessions. Authenticates with an API key (standalone) or admin certificate (plugin). |
| **Agent** | A daemon (`shell-agent serve`) running on a remote machine that spawns tmux sessions and relays terminal I/O through the WebSocket connection. |
| **Agent label** | A unique identifier for an agent, used in URLs and certificates. Format: `[a-z0-9-]+`, 1-50 chars. Example: `office-ubuntu`. |
| **Atomic write** | File write pattern (temp → fsync → rename) that prevents partial reads. Used for all JSON state files. |
| **Blocklist** | An advisory list of commands that the shell wrapper checks before execution. Not a security boundary — can be bypassed via subshells. |
| **CA (Certificate Authority)** | The root certificate used to sign agent certificates. In standalone mode, generated by the server (10-year, 4096-bit RSA). In plugin mode, Portlama's CA. |
| **Capture-pane** | tmux command (`tmux capture-pane -p -S -`) that reads the current terminal buffer. The agent runs this every 100ms. |
| **CIDR** | Classless Inter-Domain Routing notation for IP ranges. Example: `192.168.1.0/24` matches 192.168.1.0–192.168.1.255. |
| **Close code** | A numeric code sent when a WebSocket connection closes. Shell uses standard codes (1000, 1001) and custom codes (4400–4500). |
| **CSR (Certificate Signing Request)** | A request from an agent for the server to sign its public key. Generated during enrollment with `CN=agent:<label>`. |
| **Default policy** | The policy applied to agents when no specific policy is selected. Cannot be deleted. |
| **DelegatingAgentRegistry** | The agent registry adapter for plugin mode — reads/writes Portlama's agent records while preserving non-shell fields. |
| **Enrollment** | The process of registering a new agent with the server. Involves token validation, CSR signing, and credential storage. |
| **Join token** | A one-time, time-limited token used for agent enrollment. Default TTL: 10 minutes. |
| **mTLS** | Mutual TLS — both client and server present certificates during the TLS handshake. Used for agent authentication. |
| **P12 / PKCS12** | A file format that bundles a certificate, private key, and CA chain in a single password-protected file. Used in plugin mode. |
| **Pipe-pane** | tmux command (`tmux pipe-pane`) that tees all terminal I/O to a file. Used for session recording. |
| **Plugin mode** | Deployment where Shell runs as a Fastify plugin inside Portlama, using Portlama's certificates and agent registry. |
| **Policy** | A bundle of access rules: IP allowlists/denylists, command blocklist, inactivity timeout. Applied to agents when enabling access. |
| **Promise-chain mutex** | A lightweight concurrency lock that serializes async operations without blocking the event loop. |
| **Relay** | The shell server's role of forwarding WebSocket frames between admin and agent without interpreting the content. |
| **Send-keys** | tmux command for injecting keystrokes. `-l` flag sends literal characters; without it, key names are resolved (Enter, C-c, etc.). |
| **Session** | A terminal session between an admin and an agent, relayed through the server. Has a unique UUID, start/end times, and a recording. |
| **Shell wrapper** | A bash script (`shell-wrapper.sh`) that runs inside tmux, enforcing the command blocklist and logging commands. |
| **shellEnabledUntil** | An ISO 8601 timestamp on an agent record that defines when shell access expires. Set by `POST /api/shell/enable/:label`. |
| **Special key** | A named tmux key (Enter, Escape, C-c, etc.) from a fixed allowlist. Sent via the `special-key` message type. |
| **Standalone mode** | Deployment where Shell runs its own server with its own CA, API key, and agent registry. |
| **StandaloneAgentRegistry** | The file-based agent registry for standalone mode. Stores agents in `~/.shell/agents.json`. |
| **Ticket** | A single-use, short-lived (30s TTL) authorization token used in tunnel mode. Issued by the server's `TicketStore`, consumed during the WebSocket handshake on `/api/shell/agent-ticket/:label`. |
| **TicketClient** | An SDK class from `@lamalibre/portlama-tickets` used by tunnel-mode agents to poll the panel's ticket inbox, validate tickets, and send session heartbeats. |
| **TicketInstanceManager** | An SDK class from `@lamalibre/portlama-tickets` used by the server to register as a ticket instance with the Portlama panel and request tickets for agents. |
| **Time window** | The duration for which shell access is enabled for an agent. Range: 5–480 minutes. Checked at connection time and every 30 seconds during sessions. |
| **tmux** | Terminal multiplexer that manages virtual terminal sessions. Shell uses it for session management, output capture, resize, and recording. |
| **Tunnel mode** | Deployment where Shell runs its own standalone server but connects agents through Portlama's tunnel infrastructure using ticket-based auth via the `@lamalibre/portlama-tickets` SDK. |
